
## 2024-05-24 - [Fix cross-domain cookie leakage in filter_youtube_cookies]
**Vulnerability:** The `filter_youtube_cookies` function was vulnerable to cross-domain cookie leakage when parsing Netscape HTTP Cookie Files. It used simple substring matching (`"youtube.com" in line_str`), meaning a domain like `maliciousyoutube.com` would falsely trigger the domain filter and leak cookies. Additionally, it did not properly handle `#HttpOnly_` prefix which is used for valid cookies, potentially incorrectly treating them as comments.
**Learning:** Broad substring checks are dangerous when parsing structured data like Netscape HTTP Cookie Files where domain is a specific field. Furthermore, the Netscape HTTP Cookie File format abuses the comment character `#` for the `#HttpOnly_` prefix which must be evaluated as a valid cookie instead of ignored.
**Prevention:** Always split lines by tabs (`\t`) and strictly validate the domain column. Use suffix matching (e.g., `domain.endswith('.youtube.com') or domain == 'youtube.com'`) instead of strict exact match allowlists to avoid blocking valid subdomains. Check that `#HttpOnly_` is treated distinctly from standard `#` comments.
