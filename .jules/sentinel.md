## 2025-01-20 - Fix cross-domain cookie leakage in Netscape HTTP Cookie File parsers
**Vulnerability:** Naive substring matching (e.g., `"youtube.com" in line`) in cookie filtering allowed malicious or completely unrelated domains (like `youtube.com.evil.com` or `fake-youtube.com`) to leak their cookies into the filtered result. Also, `#HttpOnly_` lines were treated as comments and bypassed entirely in the original logic.
**Learning:** Always validate Netscape Cookie files strictly by separating logic into column-based parsing (`split('\t')`) and validate domains using suffix matching (`endswith`) or exact matches. `HttpOnly` cookies prefix the domain string with `#HttpOnly_`, so the parser must gracefully slice the data line to read the actual domain correctly, instead of treating it as a generic comment to be discarded or passed through without domain checks.
**Prevention:** Follow standard cookie parsing logic. Split Netscape format strings via tabs and perform exact/suffix checks against the first (domain) column. Handle `#HttpOnly_` prefixes manually since these represent secure HTTP cookies, not standard text comments.
