## 2025-02-27 - [Netscape Cookie File Parsing Vulnerability]
**Vulnerability:** The cookie sanitization logic in `filter_youtube_cookies` and `filter_vimeo_cookies` used broad substring matching (`"youtube.com" in line_str`). This could allow an attacker to bypass the filter by creating a malicious domain like `malicious-youtube.com`, leaking sensitive session tokens of the user for other sites.
**Learning:** Netscape HTTP Cookie files must be parsed structurally by splitting lines with tabs and strictly validating the domain column. Simple substring checks on raw text lines are insecure for domain validation. Furthermore, cookies with `#HttpOnly_` prefix were previously handled correctly by accident (because `#` was treated as a comment), but need to be explicitly parsed as valid cookies.
**Prevention:** Always parse structured files like Netscape HTTP Cookie files according to their specification. When validating domains, extract the domain string and use suffix matching (e.g., `domain.endswith('.youtube.com') or domain == 'youtube.com'`) instead of broad substring searches.
