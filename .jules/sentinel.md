## 2024-06-27 - [CRITICAL] Cross-Domain Cookie Leakage in Cookie Filtering
**Vulnerability:** Cookie filtering functions (`filter_youtube_cookies`, `filter_vimeo_cookies`) relied on broad substring matching (e.g., `"youtube.com" in line`) instead of strictly parsing the Netscape HTTP Cookie File format. This allowed maliciously crafted cookie entries with spoofed domains (like `evilyoutube.com`) or arbitrary user data to bypass the filter and exfiltrate user session secrets alongside valid cookies.
**Learning:** Naive substring matching on sensitive configuration/credential files is extremely dangerous and prone to bypasses via prefix/suffix manipulation. Always parse the file using its formal specification (Netscape cookie files are tab-delimited) and use strict equality or precise suffix matching for domain validation. Additionally, be aware of standard format extensions like the `#HttpOnly_` prefix which must be correctly parsed so valid secure cookies aren't ignored and treated as generic `#` comments.
**Prevention:** Always parse structured credentials like cookie files strictly according to their format (e.g., splitting by `\t`). Validate domain columns using strict domain suffix rules (e.g., `domain.endswith('.youtube.com')` or `domain == 'youtube.com'`). Include test cases for maliciously spoofed inputs (e.g., `evilyoutube.com`) and edge cases (e.g., `#HttpOnly_` prefix) to verify the robustness of credential sanitization logic.
