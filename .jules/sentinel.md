## 2025-02-28 - Insecure Cookie Domain Filtering
**Vulnerability:** The cookie filtering mechanism for YouTube/Google cookies (`filter_youtube_cookies`) used insecure partial substring matching (`"youtube.com" in line_str`). This could allow cross-domain cookie leakage, for instance from malicious domains like `evilyoutube.com`. Also, HttpOnly cookies (`#HttpOnly_.google.com`) were vulnerable to being accidentally ignored or improperly evaluated.
**Learning:** Security filters based on substrings in configuration or credential files can lead to false positives and expose critical tokens. Netscape cookie file formats require explicit tab-based parsing to accurately extract the domain column.
**Prevention:** Always parse tab-delimited or structured files (like Netscape Cookie files) column by column, handle edge cases like `#HttpOnly_`, and use strict domain matching strategies (e.g. `domain.endswith('.youtube.com')` or exact matches) rather than loosely searching the entire line.
