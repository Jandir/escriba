## 2024-11-20 - Netscape Cookie File Cross-Domain Leakage
**Vulnerability:** Substring checking (e.g., `"youtube.com" in line`) for filtering domains in Netscape Cookie Files incorrectly matches sub-domains of other services (e.g., `fakeyoutube.com`), exposing cookies across domains. Further, comments starting with `#HttpOnly_` are incorrectly treated as comments instead of valid HttpOnly cookies.
**Learning:** Netscape Cookie files format is tab-separated. Naive substring matching for domain names leaks cookies because attackers can register domains that end with or contain the target domain string. Skipping lines that start with `#` without specific handling for `#HttpOnly_` drops critical secure cookies.
**Prevention:** Always split Netscape Cookie files by tabs (`\t`), check domain parts by suffix matching (e.g., `domain.endswith('.youtube.com') or domain == 'youtube.com'`), and explicitly handle `#HttpOnly_` lines as valid cookies by stripping the prefix before processing.
