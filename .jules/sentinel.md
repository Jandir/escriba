## 2025-02-28 - Cross-Domain Cookie Leakage in Netscape Cookie File Parser
**Vulnerability:** The cookie filtering function used a broad `in` substring search (e.g., `"youtube.com" in line_str`) and failed to parse `\t`-separated columns, allowing malicious domains like `evilyoutube.com` to bypass the filter. Additionally, lines starting with `#HttpOnly_` were treated as normal comments and bypassed entirely, leaking HttpOnly cookies across all domains.
**Learning:** Netscape HTTP Cookie Files use tab-separated formats where the first column is the domain and `#` normally indicates comments, but `#HttpOnly_` is a special prefix for valid cookie lines. Simple string inclusion checks are fundamentally insecure for domain matching.
**Prevention:** Always parse cookie files correctly by splitting lines by tabs (`\t`) to isolate the domain column. Perform suffix matching (e.g., `domain.endswith('.youtube.com')` or `domain == 'youtube.com'`) instead of substring searches. Handle `#HttpOnly_` lines explicitly by stripping the prefix before parsing.
