## 2024-05-24 - [CRITICAL] Cookie Leakage via Improper Substring Matching
**Vulnerability:** In `filter_youtube_cookies` and `filter_vimeo_cookies`, cookies were filtered using a simple substring search (e.g., `"youtube.com" in line_str`). This allowed an attacker to create a malicious cookie file containing credentials for other sites (like `.facebook.com`) where the value contained the string `youtube.com` (e.g. `xyz_youtube.com_fake`), causing the script to accidentally retain and potentially leak these sensitive cookies to third parties or persist them inappropriately.
**Learning:** Never use plain substring matching to validate domain names in security contexts like cookie filtering. It is highly susceptible to bypasses and unintended inclusions.
**Prevention:** Always parse the structured format (e.g., Netscape Cookie format is tab-separated) to isolate the domain field. Use exact string matching or proper suffix matching (`.endswith(".youtube.com")`) to validate domain fields.
